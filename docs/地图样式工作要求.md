# MusicMap 地图样式调整 · 工作要求

> 负责地图样式与色彩搭配的组员。
> 视觉 / UI 设计

**所有色彩搭配都是AI生成的，仅供参考！！！ 或者不要参考，其实有点丑。**


## 目标

把 MapTiler 底图调成 **暗色、去杂物** 的"舞台",让专辑封面切片当主角;并把 **红色作为这张地图的记忆色**(近网易云红),像"绿=自然"一样,让人一看就联想到我们。

**结构、字体、标注处理参照样式参考图,配色换成以红色为主色调以及其他搭配的色彩的组合。** 核心:底图要"退后",凡是和"专辑 / 国家 / 大洲"无关的要素都弱化或隐藏。

---

## 1. 配色 · 红色记忆锚点

红色 **常驻** 铺在国家面上(见第 2 条),让整张图基调就是暗红——这就是记忆点。关键用 **深、降饱和的红**,不要大面积亮红,这样既有识别度又不突兀、不跟封面打架。

建议色值(仅供参考,AI生成的配色；**相信你的审美**):

| 用途 | 备选色值 | 说明 |
|------|----------|------|
| 红 · 主(强调) | `#C20C0C` / 提亮版 `#E63329` | 近网易云红,用于线、UI 强调 |
| 红 · 国家面常驻 | `#2A1216` ~ `#3A1820` | 深酒红 / 砖红,大面积铺也不刺眼 |
| 网格线 | `rgba(194,12,12,0.30)` 暗红 **或** `rgba(184,144,64,0.25)` 古铜金 | 二选一,试哪个配封面更顺;主打红时倾向暗红 |
| 海洋 / 背景 | `#0A0E14`(近黑微冷) | 和陆地拉开层次 |

## 2. 国家面颜色 + 矢量显示策略

**国家面 (fill) —— 常驻显示、常驻上色:**

- 全部国家面 **始终填充** 既透出封面切片,又让全图维持红色基调。
- 国界线保留、并设置相对清晰的国界线，参考样图。

**需要隐去的底物(专辑地图用不到):**

- ❌ 道路 / 交通
- ❌ 兴趣点 POI、小地名
- ❌ 建筑
- ❌ 河流水系、湖泊(或保留湖泊面但调到极暗)
- ❌ 首都 / 城市圆点及城市名

**保留:** 国家面(常驻红)、国界线、海岸线(可弱)、**大洲 / 海洋 / 国家** 标注。

---

## 3. 字体 / 标注(用英文，显得显得没有母语羞耻)



**通用要求:**

- 标注语言 **强制英文**:每个 label 图层的 text-field 改成 `{name:latin}` / `{name_en}`,不要跟随本地语言。
- 字体在 MapTiler 字体库里选:大洲 / 海洋挑一款有气质的(衬线或宽字距无衬线),国家用干净无衬线;先各试 1–2 款对比。
- 字会压在花花绿绿的封面切片上,没描边会糊。看看描边配色。

## 交接方式(重要)

**组员在自己的 MapTiler 账号里调样式,把样式 URL 直接发给组长即可,不用导出文件。**

1. 组员用自己的(免费)MapTiler 账号新建 / 编辑这张地图。
2. 调好后,把样式的 URL 发给组长:
   ```
   https://api.maptiler.com/maps/<你的样式ID>/style.json?key=<你的key>
   ```
   **就是USE VECTOR STYLE**
3. 组长把前端 `frontend/grid-inspector.html` 里这两个常量换成组员的值即可:
   ```js
   const MAPTILER_KEY = '<组员的key>';
   const STYLE_ID     = '<组员的样式ID>';
   ```
4. **实时同步**:此后组员每次在自己账号里改完保存,组长刷新页面就能看到最新样式,不用反复发文件。
